# 算子的多头分块计算详解

## 1. 整体架构和张量维度

### 输入张量形状

```
q: (B, T, H,  K)   - Query  (B个样本, T个时间步, H个头, K维特征)
k: (B, T, H,  K)   - Key    (B个样本, T个时间步, H个头, K维特征)
v: (B, T, HV, V)   - Value  (B个样本, T个时间步, HV个头, V维特征)
o: (B, T, HV, V)   - Output (B个样本, T个时间步, HV个头, V维特征)

重要关系：
  H  = Query/Key 的头数 (例如: 32)
  HV = Value 的头数     (例如: 32 或 8)

  如果 H == HV: 标准多头注意力
  如果 H > HV:  Grouped Query Attention (GQA)
                例如 H=32, HV=8, 每 4 个 q/k 头共享 1 个 v 头
```

### 隐藏状态

```
h: (N, HV, K, V)  - 每个 batch、每个 value head 都有一个 K×V 的隐藏状态矩阵

初始化: h = zeros(K, V)  (针对每个 batch 和 value head)
每个时间步更新 h
```

## 2. Grid 配置和分块策略

### Grid 三维结构

```python
BK = next_power_of_2(K)      # K 维度块大小 (通常 = K = 64)
BV = min(next_power_of_2(V), 64)  # V 维度块大小 (8 或 64)
NK = cdiv(K, BK)             # K 维度块数量 (通常 = 1)
NV = cdiv(V, BV)             # V 维度块数量 (例如 V=128, BV=64 → NV=2)

grid = (NK, NV, N * HV)      # 三维 Grid
       ↓   ↓   ↓
       │   │   └─ Batch × Value Heads (例如: 8 batches × 32 heads = 256)
       │   └───── V 维度分块 (例如: 2 blocks)
       └───────── K 维度分块 (通常: 1 block)

总 Block 数 = NK × NV × N × HV
```

### Grid 可视化 (假设 N=2, HV=4, NK=1, NV=2)

```
3D Grid 结构:
═══════════════════════════════════════════════════════════════

维度 0 (i_k): K 分块
    │
    └─→ [0]  (通常只有一个，因为 K 不大)

维度 1 (i_v): V 分块
    │
    ├─→ [0]  处理 V 的前半部分 (v[0:64])
    └─→ [1]  处理 V 的后半部分 (v[64:128])

维度 2 (i_nh): Batch × Value Heads (联合索引)
    │
    ├─→ [0]  = Batch 0, Value Head 0
    ├─→ [1]  = Batch 0, Value Head 1
    ├─→ [2]  = Batch 0, Value Head 2
    ├─→ [3]  = Batch 0, Value Head 3
    ├─→ [4]  = Batch 1, Value Head 0
    ├─→ [5]  = Batch 1, Value Head 1
    ├─→ [6]  = Batch 1, Value Head 2
    └─→ [7]  = Batch 1, Value Head 3

总 blocks = 1 × 2 × 8 = 16 blocks
```

## 3. Program ID 到索引的映射

### 从 Grid 坐标解码到实际索引

```python
# Grid 坐标
i_k  = tl.program_id(0)  # K 维度块索引 [0, NK)
i_v  = tl.program_id(1)  # V 维度块索引 [0, NV)
i_nh = tl.program_id(2)  # Batch × ValueHead 联合索引 [0, N*HV)

# 解码
i_n  = i_nh // HV        # Batch 索引
i_hv = i_nh % HV         # Value Head 索引
i_h  = i_hv // (HV // H) # Query/Key Head 索引

# 元素偏移
o_k = i_k * BK + tl.arange(0, BK)  # [i_k*BK, ..., i_k*BK+BK-1]
o_v = i_v * BV + tl.arange(0, BV)  # [i_v*BV, ..., i_v*BV+BV-1]
```

### 映射示例

```
假设: H=32, HV=32 (标准多头注意力)

Block (i_k=0, i_v=0, i_nh=0):
  i_n  = 0 // 32 = 0        → Batch 0
  i_hv = 0 % 32  = 0        → Value Head 0
  i_h  = 0 // 1  = 0        → Query/Key Head 0
  处理: Batch 0, Head 0, K[0:64], V[0:64]

Block (i_k=0, i_v=1, i_nh=0):
  i_n  = 0 // 32 = 0        → Batch 0
  i_hv = 0 % 32  = 0        → Value Head 0
  i_h  = 0 // 1  = 0        → Query/Key Head 0
  处理: Batch 0, Head 0, K[0:64], V[64:128]

Block (i_k=0, i_v=0, i_nh=33):
  i_n  = 33 // 32 = 1       → Batch 1
  i_hv = 33 % 32  = 1       → Value Head 1
  i_h  = 1 // 1   = 1       → Query/Key Head 1
  处理: Batch 1, Head 1, K[0:64], V[0:64]
```

### Grouped Query Attention 示例

```
假设: H=32, HV=8 (4 个 q/k 头共享 1 个 v 头)

Block (i_k=0, i_v=0, i_nh=0):
  i_n  = 0 // 8 = 0         → Batch 0
  i_hv = 0 % 8  = 0         → Value Head 0
  i_h  = 0 // (8/32) = 0 // 4 = 0  → Query/Key Head 0

  注意: Value Head 0 对应 Query/Key Heads [0, 1, 2, 3]
        但这个 block 只处理 Head 0 的 q/k
```

## 4. 每个 Block 的数据访问

### 4.1 指针计算

```python
# 时间范围 (针对当前 batch)
bos = i_n * T              # 起始时间步
eos = (i_n + 1) * T        # 结束时间步

# Query/Key 指针 (使用 i_h - q/k head 索引)
p_q = q + (bos * H + i_h) * K + o_k
      ↑     ↑        ↑      ↑     ↑
      │     │        │      │     └─ K 维度偏移 [i_k*BK : i_k*BK+BK]
      │     │        │      └─────── K 维度跨步
      │     │        └────────────── Head 索引 (q/k head)
      │     └─────────────────────── 时间步偏移
      └───────────────────────────── 基地址

# Value 指针 (使用 i_hv - value head 索引)
p_v = v + (bos * HV + i_hv) * V + o_v
      ↑     ↑         ↑       ↑     ↑
      │     │         │       │     └─ V 维度偏移 [i_v*BV : i_v*BV+BV]
      │     │         │       └─────── V 维度跨步
      │     │         └───────────────Head 索引 (value head)
      │     └─────────────────────────时间步偏移
      └───────────────────────────────基地址
```

### 4.2 数据访问示意图

```
示例: Block (i_k=0, i_v=0, i_nh=0)
     → Batch 0, Head 0, K[0:64], V[0:64]

═══════════════════════════════════════════════════════════════

Query 张量: q[B=8, T=100, H=32, K=64]
                    ↓ 访问这个切片
    ┌─────────────────────────────────────┐
    │ Batch 0, t=0, Head 0, K[0:64]       │ ← 第一个时间步
    │ Batch 0, t=1, Head 0, K[0:64]       │ ← 第二个时间步
    │ Batch 0, t=2, Head 0, K[0:64]       │
    │ ...                                 │
    │ Batch 0, t=99, Head 0, K[0:64]      │ ← 最后时间步
    └─────────────────────────────────────┘
     共 100 个时间步，每个加载 64 个元素

Key 张量: k[B=8, T=100, H=32, K=64]
                    ↓ 访问这个切片 (和 q 相同)
    ┌─────────────────────────────────────┐
    │ Batch 0, t=0, Head 0, K[0:64]       │
    │ Batch 0, t=1, Head 0, K[0:64]       │
    │ ...                                 │
    └─────────────────────────────────────┘

Value 张量: v[B=8, T=100, HV=32, V=128]
                    ↓ 访问这个切片
    ┌─────────────────────────────────────┐
    │ Batch 0, t=0, Head 0, V[0:64]       │ ← 只访问前 64 个
    │ Batch 0, t=1, Head 0, V[0:64]       │    (因为 i_v=0)
    │ ...                                 │
    └─────────────────────────────────────┘
     共 100 个时间步，每个加载 64 个元素

隐藏状态: h[K=64, V=64] (本地寄存器)
    ┌───────────────────────────┐
    │                           │
    │   64 × 64 矩阵            │
    │   存储在寄存器/shared mem │
    │                           │
    └───────────────────────────┘
     在所有时间步之间持续更新
```

## 5. 完整的多头分块计算示意图

### 5.1 单个 Batch 的多头处理 (标准多头注意力)

```
假设: N=1, H=4, HV=4, K=64, V=128, BK=64, BV=64
     → NK=1, NV=2

Grid = (1, 2, 1×4) = 总共 8 个 blocks

═══════════════════════════════════════════════════════════════
                       Batch 0
═══════════════════════════════════════════════════════════════

Head 0 (i_hv=0):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,0)       │ Block (0,1,0)       │
│ Head 0, V[0:64]     │ Head 0, V[64:128]   │
│                     │                     │
│ 处理:               │ 处理:               │
│ - q[0,:,0,:]        │ - q[0,:,0,:]  (相同)│
│ - k[0,:,0,:]        │ - k[0,:,0,:]  (相同)│
│ - v[0,:,0,0:64]     │ - v[0,:,0,64:128]   │
│ - h[0:64, 0:64]     │ - h[0:64, 64:128]   │
└─────────────────────┴─────────────────────┘

Head 1 (i_hv=1):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,1)       │ Block (0,1,1)       │
│ Head 1, V[0:64]     │ Head 1, V[64:128]   │
│                     │                     │
│ 处理:               │ 处理:               │
│ - q[0,:,1,:]        │ - q[0,:,1,:]  (相同)│
│ - k[0,:,1,:]        │ - k[0,:,1,:]  (相同)│
│ - v[0,:,1,0:64]     │ - v[0,:,1,64:128]   │
│ - h[0:64, 0:64]     │ - h[0:64, 64:128]   │
└─────────────────────┴─────────────────────┘

Head 2 (i_hv=2):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,2)       │ Block (0,1,2)       │
│ Head 2, V[0:64]     │ Head 2, V[64:128]   │
└─────────────────────┴─────────────────────┘

Head 3 (i_hv=3):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,3)       │ Block (0,1,3)       │
│ Head 3, V[0:64]     │ Head 3, V[64:128]   │
└─────────────────────┴─────────────────────┘

关键观察:
1. 每个 head 独立处理 (并行)
2. 每个 head 的 V 维度被分成 2 个 blocks (串行或并行)
3. 同一 head 的不同 V blocks 处理相同的 q/k，但不同部分的 v
4. 不同 heads 之间完全独立，无数据共享
```

### 5.2 V 维度分块的详细说明

```
Head 0 的处理 (V=128, BV=64):

完整的 h 矩阵 (K×V = 64×128):
┌─────────────────────────────────────────┐
│                                         │ K=64
│              h 矩阵                     │
│                                         │
└─────────────────────────────────────────┘
              V=128

分块处理:
┌───────────────────┬───────────────────┐
│   Block (0,0,0)   │   Block (0,1,0)   │
│   h[:,0:64]       │   h[:,64:128]     │ K=64
│                   │                   │
└───────────────────┴───────────────────┘
     V[0:64]             V[64:128]

每个 Block 的操作 (伪代码):
for t in range(T):
    # Block (0,0,0)
    b_q = load q[0, t, 0, :]          # (64,)
    b_k = load k[0, t, 0, :]          # (64,)
    b_v = load v[0, t, 0, 0:64]       # (64,)  ← 只加载前半

    b_h[:,0:64] = update(b_h[:,0:64], b_q, b_k, b_v)
    b_o = compute_output(b_h[:,0:64], b_q)
    store o[0, t, 0, 0:64] = b_o

    # Block (0,1,0)
    b_q = load q[0, t, 0, :]          # (64,)  ← 同样的 q
    b_k = load k[0, t, 0, :]          # (64,)  ← 同样的 k
    b_v = load v[0, t, 0, 64:128]     # (64,)  ← 加载后半

    b_h[:,64:128] = update(b_h[:,64:128], b_q, b_k, b_v)
    b_o = compute_output(b_h[:,64:128], b_q)
    store o[0, t, 0, 64:128] = b_o
```

## 6. Grouped Query Attention (GQA) 的特殊情况

### 6.1 配置示例

```
H=32 (32 个 q/k 头)
HV=8 (8 个 v 头)
每 4 个 q/k 头共享 1 个 v 头

映射关系:
Value Head 0 ← Query/Key Heads [0, 1, 2, 3]
Value Head 1 ← Query/Key Heads [4, 5, 6, 7]
...
Value Head 7 ← Query/Key Heads [28, 29, 30, 31]
```

### 6.2 Head 映射计算

```python
# 在 kernel 中
i_hv = i_nh % HV         # Value head index [0, 8)
i_h  = i_hv // (HV // H) # Query/Key head index
     = i_hv // (8 // 32)
     = i_hv // 4

示例:
i_hv = 0 → i_h = 0  (使用 q/k head 0)
i_hv = 1 → i_h = 4  (使用 q/k head 4)
i_hv = 2 → i_h = 8  (使用 q/k head 8)
...
```

### 6.3 GQA 的 Block 分配

```
假设: N=1, H=4, HV=2, K=64, V=128, BK=64, BV=64
     → NK=1, NV=2
     → 每 2 个 q/k 头共享 1 个 v 头

Grid = (1, 2, 1×2) = 总共 4 个 blocks

═══════════════════════════════════════════════════════════════

Value Head 0 (对应 Query/Key Heads 0-1):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,0)       │ Block (0,1,0)       │
│ i_hv=0, i_h=0       │ i_hv=0, i_h=0       │
│                     │                     │
│ 使用:               │ 使用:               │
│ - q/k head 0        │ - q/k head 0  (相同)│
│ - v head 0 (前半)   │ - v head 0 (后半)   │
└─────────────────────┴─────────────────────┘

Value Head 1 (对应 Query/Key Heads 2-3):
┌─────────────────────┬─────────────────────┐
│ Block (0,0,1)       │ Block (0,1,1)       │
│ i_hv=1, i_h=2       │ i_hv=1, i_h=2       │
│                     │                     │
│ 使用:               │ 使用:               │
│ - q/k head 2        │ - q/k head 2  (相同)│
│ - v head 1 (前半)   │ - v head 1 (后半)   │
└─────────────────────┴─────────────────────┘

注意:
1. Value head 0 只和 q/k head 0 交互 (不是 head 1)
2. 实际的 "grouping" 发生在 Attention 层的其他部分
3. 这个 kernel 处理的是单个 value head 对应的单个 q/k head
```

## 7. 时间循环中的数据流

### 单个 Block 的完整时间循环

```
Block (i_k=0, i_v=0, i_nh=0)
处理: Batch 0, Head 0, K[0:64], V[0:64]

═══════════════════════════════════════════════════════════════

初始化:
  h[64×64] = zeros  (或从 initial_state 加载)

for t in 0 to T-1:
    ┌─────────────────────────────────────────────────────────┐
    │ 时间步 t                                                │
    ├─────────────────────────────────────────────────────────┤
    │                                                         │
    │ 1. Load 输入 (从 global memory)                         │
    │    ┌──────────────────────────────────┐                │
    │    │ b_q  = q[0, t, 0, 0:64]   (64,) │                │
    │    │ b_k  = k[0, t, 0, 0:64]   (64,) │                │
    │    │ b_v  = v[0, t, 0, 0:64]   (64,) │                │
    │    │ b_b  = b[0, t, 0]         (1,)  │                │
    │    │ b_a  = a[0, t, 0]         (1,)  │                │
    │    └──────────────────────────────────┘                │
    │                                                         │
    │ 2. 计算门控参数                                         │
    │    g = -exp(A_log) * softplus(a + dt_bias)             │
    │    beta = sigmoid(b)                                    │
    │                                                         │
    │ 3. 更新隐藏状态 (在寄存器中)                            │
    │    ┌─────────────────────────────────┐                 │
    │    │ h = h * exp(g)          # 衰减  │                 │
    │    │ v' = v - sum(h * k)     # Delta │                 │
    │    │ v' = v' * beta          # 门控  │                 │
    │    │ h = h + k ⊗ v'          # 更新  │                 │
    │    └─────────────────────────────────┘                 │
    │      h[64×64] 矩阵在寄存器中持续更新                   │
    │                                                         │
    │ 4. 计算输出                                             │
    │    ┌─────────────────────────────────┐                 │
    │    │ o = sum(h * q, dim=0)    (64,) │                 │
    │    └─────────────────────────────────┘                 │
    │                                                         │
    │ 5. Store 输出 (到 global memory)                        │
    │    o[0, t, 0, 0:64] = o                                │
    │                                                         │
    └─────────────────────────────────────────────────────────┘

    h 在循环中传递 →

结束后:
  可选: 存储最终 h 到 initial_state (供下次使用)

═══════════════════════════════════════════════════════════════

关键点:
1. h 矩阵常驻寄存器，不需要每次从内存加载
2. q, k, v 每个时间步从 global memory 加载
3. 输出 o 每个时间步写回 global memory
4. 时间步之间串行执行 (因为有依赖关系)
```

## 8. 内存访问模式详解

### 8.1 Strided Access Pattern

```
Query 张量布局: q[B, T, H, K]

连续内存:
[B=0,T=0,H=0,K=0...63][B=0,T=0,H=1,K=0...63]...[B=0,T=1,H=0,K=0...63]...

Block (0,0,0) 的访问模式 (i_n=0, i_h=0):

t=0:  Load q[0, 0, 0, 0:64]   地址: base + 0*H*K + 0*K
                                     ↓
t=1:  Load q[0, 1, 0, 0:64]   地址: base + 1*H*K + 0*K
                                     ↓ 跨步 = H*K
t=2:  Load q[0, 2, 0, 0:64]   地址: base + 2*H*K + 0*K
                                     ↓
...

跨步访问，但每次访问是连续的 64 个元素 (coalesced)
```

### 8.2 不同 Blocks 的内存访问

```
假设同时运行 Block (0,0,0) 和 Block (0,0,1):

时间步 t=0:

Block (0,0,0):               Block (0,0,1):
Load q[0,0,0,0:64]          Load q[0,0,1,0:64]
     ↑                           ↑
     地址 A                      地址 A + K

Load k[0,0,0,0:64]          Load k[0,0,1,0:64]
Load v[0,0,0,0:64]          Load v[0,0,1,0:64]

不同 heads 访问不同的内存区域 → 无内存冲突
可以完全并行执行
```

## 9. 性能关键因素

### 9.1 为什么 V 维度分块很重要

```
完整 h 矩阵: 64 × 128 = 8192 elements
每个 element 4 bytes (float32)
总大小: 32 KB

GPU 寄存器限制:
  每个线程: ~255 registers
  每个 warp (32 threads): ~8K registers
  如果 num_warps=4: ~32K registers

如果不分块 (BV=128):
  需要存储整个 h[64×128] = 8192 elements
  如果分配到 128 threads (4 warps):
    每个线程: 8192/128 = 64 elements
    需要 64 个寄存器 (可行，但紧张)

分块后 (BV=64):
  只需要存储 h[64×64] = 4096 elements
  如果分配到 128 threads:
    每个线程: 4096/128 = 32 elements
    需要 32 个寄存器 (宽松)

优势:
✅ 降低寄存器压力
✅ 允许更多 warps 并发
✅ 提高 SM 占用率
```

### 9.2 Block 并行度

```
原始 (BV=8):
  N=4, HV=32, NV=16 (V=128, BV=8)
  总 blocks = 1 × 16 × 4 × 32 = 2048 blocks

  高并行度 ✅
  但每个 block 工作量小 ❌
  调度开销大 ❌

优化后 (BV=64):
  N=4, HV=32, NV=2 (V=128, BV=64)
  总 blocks = 1 × 2 × 4 × 32 = 256 blocks

  中等并行度 (仍然足够) ✅
  每个 block 工作量大 ✅
  调度开销低 ✅
```

## 10. 总结

### 分块策略

```
1. Batch 维度 (N):     每个 block 处理一个 batch
2. Value Head 维度 (HV): 每个 block 处理一个 head
3. K 维度:             通常不分块 (NK=1)
4. V 维度:             分块 (NV = cdiv(V, BV))

关键设计:
- 不同 batches、不同 heads 之间完全独立并行
- 同一 head 的不同 V blocks 可以并行 (独立计算)
- 时间维度 (T) 串行处理 (因为有递归依赖)
```

### 数据流

```
输入:
  q, k ← 根据 i_h (q/k head) 选择
  v    ← 根据 i_hv (value head) 选择

中间状态:
  h ← 存储在寄存器中，跨时间步传递

输出:
  o ← 每个时间步写回，位置根据 i_hv 确定
```

### 性能关键

```
✅ V 维度分块 → 降低寄存器压力
✅ BV 增大 → 减少 blocks，降低调度开销
✅ num_warps 增大 → 匹配更大的工作量
✅ 多头并行 → 充分利用 GPU
✅ h 常驻寄存器 → 避免重复内存访问
```
